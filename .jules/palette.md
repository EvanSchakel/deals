## 2024-05-24 - Interactive Form Clarity
**Learning:** CLI tool users benefit from explicit visual cues for optional fields, just as much as GUI users do. Using dimmed text for `(opt)` indicators is an effective way to guide the user without adding cognitive load or disrupting layout alignment.
**Action:** When working on CLI forms, ensure optional fields are explicitly marked and leverage existing color/formatting utilities (like `Color.DIM`) for visual hierarchy. Maintain column alignment by ensuring the visual length of prompts with colors are accounted for.
## 2024-05-24 - CLI Actionable Errors and Shortcut Hints
**Learning:** For terminal applications without visual context, users often get stuck on validation errors (like "Could not parse price"). Explicit actionable examples (e.g., "Please enter a number, e.g., 1200 or 1200.50") drastically reduce friction. Additionally, when exposing keyboard shortcuts via brackets (e.g., `[s]core`), the backend alias logic (`if cmd in ('score', 's')`) often already exists but isn't effectively communicated to the user.
**Action:** When improving CLI prompts, verify the backend routing logic first to see if hidden shortcuts exist, then expose them visually via brackets in all relevant menus (primary prompt, help, and error fallbacks) without needing to alter core control flow.
